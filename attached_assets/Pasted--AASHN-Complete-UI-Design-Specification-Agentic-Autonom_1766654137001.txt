## 1. Design Philosophy & Principles {#design-philosophy}

### Core Design Principles

1. **Information Hierarchy**: Critical information (alerts, incidents) always visible
2. **Real-Time First**: Live updates without page refresh (WebSocket/SSE)
3. **Contextual Awareness**: Show relevant data based on user role and current incident
4. **Progressive Disclosure**: Detailed views available on demand, not cluttered
5. **Action-Oriented**: Clear CTAs for manual overrides and approvals
6. **Trust & Transparency**: Show agent reasoning, confidence scores, audit trails

### User Personas

- **Network Operator**: Primary user, monitors incidents, approves remediations
- **Network Architect**: Reviews RCA, validates learning patterns
- **SRE Manager**: Views KPIs, system health, compliance reports
- **Compliance Officer**: Accesses audit trails, policy enforcement logs

### 2.2 Navigation Structure

**Primary Navigation (Sidebar, 240px width)**
- **Dashboard** (`/dashboard`) - Main overview
- **Incidents** (`/incidents`) - Active and historical incidents
- **Devices** (`/devices`) - Device inventory and details
- **Metrics** (`/metrics`) - Detailed metric analysis
- **Topology** (`/topology`) - Network graph visualization
- **Agents** (`/agents`) - Agent status and execution logs
- **Audit Trail** (`/audit`) - Compliance and action logs
- **Settings** (`/settings`) - Configuration and policies

**Secondary Navigation (Breadcrumbs)**
```
Home > Incidents > INC-2025-001 > Details
```

**Quick Actions (Floating Action Button)**
- Manual Remediation Trigger
- Create Custom Alert Rule
- Export Report
- Switch User Context

## 5. Real-Time Data Visualization {#real-time-viz}

### 5.1 WebSocket Integration

**Connection**: `wss://aashn.example.com/ws`

**Message Types**:
```typescript
interface WSMessage {
  type: 'metric_update' | 'alert' | 'incident' | 'agent_status';
  payload: any;
  timestamp: string;
}
```

**Update Strategy**:
- **Metrics**: Batch updates every 5 seconds
- **Alerts**: Immediate push on creation
- **Incidents**: Push on status change
- **Agent Status**: Push on state transition

### 5.2 Data Refresh Indicators

**Visual Indicators**:
- **Last Updated Timestamp**: "Updated 2s ago" (green when <10s, yellow <30s, red >30s)
- **Loading Spinner**: Show during data fetch
- **Connection Status**: Badge in header (ðŸŸ¢ Connected / ðŸ”´ Disconnected)

### 5.3 Optimistic Updates

**Strategy**:
- Show predicted state immediately
- Update with actual data when received
- Rollback if prediction incorrect

## 10. Accessibility & Performance {#accessibility}

### 10.1 Accessibility (WCAG 2.1 AA)

**Requirements**:
- **Keyboard Navigation**: All features accessible via keyboard
- **Screen Reader Support**: ARIA labels, semantic HTML
- **Color Contrast**: Minimum 4.5:1 for text
- **Focus Indicators**: Visible focus rings
- **Alt Text**: All images have descriptive alt text

### 10.2 Performance Targets

**Metrics**:
- **First Contentful Paint**: < 1.5s
- **Time to Interactive**: < 3s
- **Largest Contentful Paint**: < 2.5s
- **Cumulative Layout Shift**: < 0.1

**Optimizations**:
- **Code Splitting**: Route-based and component-based
- **Lazy Loading**: Images and charts
- **Virtual Scrolling**: For long lists
- **Memoization**: React.memo, useMemo for expensive computations

### 10.3 Error Handling

**Error States**:
- **Network Errors**: Retry button, error message
- **Empty States**: Helpful messages, CTAs
- **Loading States**: Skeleton screens, spinners
- **404 Pages**: Friendly error page with navigation
